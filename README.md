# 伴讀 bandu

**讓「想陪孩子讀書的大人」和「需要陪伴的孩子」之間，有一條安全的路。**

這是一個構想階段的公益專案。目標不是再做一個學習平台——台灣的免費學習內容已經夠多了（均一、因材網、PaGamO）——而是解決真正稀缺的東西：**人的陪伴**。我們想做的是遠距伴讀志工的媒合與營運基建，讓下了班的社會人士也能安全地陪偏鄉或弱勢的孩子讀一本書、寫一份作業。

> 現況：調研與設計階段，還沒有任何程式碼。所有開放問題都在 [Issues](../../issues) 裡，歡迎任何人參與討論——特別歡迎兒少保護、教育現場、非營利營運背景的朋友指教。

## 為什麼做這個

我們先做了功課，結論如下（附出處）：

1. **硬體與內容的落差已大致被解決。**「班班有網路、生生用平板」四年投入 200 億、61 萬台平板，偏遠學校一生一機（[行政院](https://www.ey.gov.tw/Page/448DE008087A1971/0ba98487-02aa-4bf3-afa5-89a628e29c9d)）。免費教材過剩。
2. **但落差搬家了：搬到「人」身上。** 桃園審計發現大量學校平板開機率不到五成，多數是偏遠學校——平板鎖在充電車裡（[聯合報](https://udn.com/news/story/6885/8225132)）。研究一再指出弱勢孩子缺的是學習動機與身邊的大人，不是內容（[報導者](https://www.twreporter.org/a/covid-19-education-opportunity-inequality)）。
3. **陪伴類計畫有效但量太小。** 教育部[數位學伴](https://depart.moe.edu.tw/ED2700/News_Content.aspx?n=727087A8A1328DEE&s=52FEAB3043C37A59)跑了近 20 年、累計僅服務約 1.9 萬名學童，且只收大學生當志工。均一自己的數據也顯示：平台在「有大人帶著用」的場景才有顯著成效（[2024 學習扶助追蹤](https://si.taiwan.gov.tw/Home/Org?Fid=405)）。
4. **既有的遠距伴讀計畫各自為政。** [誠致 KIST 學伴](https://www.chengzhiedu.org/partner-career/reading-buddy/)（收社會人士）、[台灣微客數位伴讀](https://www.waker.org.tw/Location.aspx?tid=75&id=56)、[T22 線上學伴](https://www.gemt.org.tw/t22%E4%B8%AD%E5%B0%8F%E5%AD%B8%E6%95%99%E8%82%B2%E8%81%AF%E7%9B%9F)等都在做遠距陪伴，但各用表單＋LINE 營運，沒有共用的基建，社會人士想投入也不容易找到入口。

所以縫隙不是「另一個平台給孩子用」，而是**給「做陪伴的人與機構」的基礎建設**：志工招募、篩選流程、排程、時數與陪伴紀錄、安全機制。

## 安全優先（這是底線，不是功能）

兒少安全決定這個專案的一切設計。目前的立場：

- **不做無中介的陌生人配對。** 台灣沒有可公開查詢的性犯罪紀錄制度；法定的不適任查核只有立案機構有權進行。因此篩選與監督必須由**機構端**（課輔據點、學校、立案 NGO）承擔，平台做的是讓這個流程更容易，而不是取代它。
- **不自建視訊。** 使用成熟工具（Google Meet／Jitsi 等），不經手、不儲存孩子的影像。
- **兒少個資最小化。** 平台盡可能不持有孩子的個資；孩子端的身分由機構管理。
- **媒介的安全取捨是開放問題。** 在家視訊、純語音、遊戲內共讀，各有不同的風險結構——這題我們要請教專業（台灣展翅協會、iWIN），見 Issues。

已知的核心張力：**社會人士志工下班後才有空，但孩子若在據點端上線，據點的開放時段有限**。純機構模式安全但時段受限；在家模式彈性但安全設計難度高。這是本專案要解的第一道真正的題目。

若你就是上述領域的工作者，或認識願意給建議的人，請開一個 issue 或在任何 issue 留言，我們會非常感激。

## 這個專案不是什麼

- 不是學習內容平台（均一們已經做得很好）
- 不是家教媒合生意（本專案不以營利為目的）
- 不是「用 app 取代陪伴」——工具只能支援陪伴者，不能取代他們
